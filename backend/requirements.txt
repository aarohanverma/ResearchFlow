# ─── PDF Parsing ──────────────────────────────────────────────────────────────
# Default parser: Marker (PDF_PARSER=marker). Runtime fallback chain:
#   <chosen> → Marker (always in chain) → Gemini Vision → minimal placeholder
#
# Docling + EasyOCR are NOT in this base requirements file anymore — they
# pull in PyTorch + OpenCV (~2 GB image bloat) and were only used when the
# operator explicitly set ``PDF_PARSER=docling``. The parser still works:
# the ``docling_parser`` adapter handles ``ImportError`` and surfaces a
# clear "install docling to use this parser" message.
#
# To re-enable docling on a host that needs it:
#     pip install docling==2.93.0 easyocr==1.7.2 \
#         --extra-index-url https://download.pytorch.org/whl/cpu
#
# Marker (the default) still pulls in torch via Surya — that's the
# dominant size in this image. Replacing it requires switching the
# default parser to Gemini Vision (no local model needed) — a runtime
# behavior change, so it is intentionally NOT done here.
marker-pdf==0.3.10         # DEFAULT parser — strong on two-column academic layouts, memory-safe
pymupdf==1.27.2.3          # required by GeminiVisionFallback for page rendering
